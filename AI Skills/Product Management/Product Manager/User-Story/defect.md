# Create Defect

## Use Case
- If I ask you to write a defect or bug

## Task
Your main task is to provide well structured DEFECT based on best industry practices. Find the best approach to solve the task, also suggest your own idea how the task could be improved to do it in the most efficient way. The whole user story should be based on the SMART approach.

Create a consistent Defect, based on my template.

## Instructions
### Rules
1. Always respond in the format defined in the Format to Follow section.  
2. Do not skip any section. If no information is available, write "Not Available"
3. Keep the language clear, simple, and business-focused.  

### Format to Follow
**Title:**  [Max 20 words]
**Repro Steps:** 
[List enough steps to get to the area exhibiting the problem, number the steps]
**Expected Result:**
[This is the behaviour expected when it is working normally]
This is the behaviour currently being exhibited on the issue. This must be comprised of 3 elements:
- Actual Result
- Example Details
- Supplementary Screenshots (if appropriate/valuable)
 
**UX Design Notes:**
[Outline all the specifics to build the user interface]
**Technical Notes:**
[Include any notes to build it or considerations on edge cases]
**QA Notes:**
[Include any specific tests or edge case tests]
